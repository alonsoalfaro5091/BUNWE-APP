¿Qué es JSON? — El idioma común de la app
📦 JSON = paquete de datos entre Flutter y Python
Como un diccionario Python que viaja por la red
Así se ve el JSON de sueño
{
  "puntuacion": 85,
  "horas": 7.5,
  "racha_dias": 5,
  "ultima_noche": "2025-06-10",
  "historial": [
    { "dia": "Lun", "horas": 7.0 },
    { "dia": "Mar", "horas": 6.5 },
    { "dia": "Mie", "horas": 8.0 }
  ]
}
Cómo funciona
→Idéntico a un diccionario Python: clave + valor
→P4 (Python) lo genera · P3 (Flutter) lo recibe y pinta
→Es texto plano — podés verlo en el navegador con la IP de la Pi
→{ } = un objeto · [ ] = lista de objetos
→P4 y P3 lo acuerdan antes de escribir una línea de código

>> [ SECCIÓN 6: DESHACER CAMBIOS ]
Para cuando algo sale mal. Leer con cuidado — algunos son reversibles y otros no. Nunca hacer reset en commits que ya subiste con push.
COMANDO
UTILIDAD / QUÉ HACE
MODO DE USO / EJEMPLO
git restore <archivo>
Descarta los cambios de UN archivo y lo vuelve al estado del último commit. Irreversible.
git restore backend/main.py
git restore .
Descarta TODOS los cambios no commiteados en la carpeta actual. Vuelve todo al último commit.
git restore .
git reset HEAD~1
Deshace el último commit pero conserva los archivos modificados. Los cambios quedan sin commitear.
git reset HEAD~1
git revert <id-commit>
Crea un nuevo commit que deshace los cambios de uno anterior. Seguro porque no borra historial.
git revert a3f5b2c
git stash
Guarda tus cambios actuales en un cajón temporal sin hacer commit. Limpia tu carpeta para cambiar de rama.
git stash
git stash pop
Recupera los cambios guardados con stash y los aplica en la rama actual.
git stash pop
git stash list
Lista todos los stashes guardados en caso de que hayas guardado varios.
git stash list

1Lógica de puntuación de sueño
La lógica de negocio va en Python — no en Flutter. Ejemplo simple:
Función Python
def calcular_puntuacion(horas: float) -> int:
    """8h = 100pts. Se resta 10 por cada hora de diferencia."""
    diferencia = abs(horas - 8.0)
    return max(0, 100 - int(diferencia * 10))
2POST /guardar completo
Endpoint que calcula y guarda
@app.post("/guardar")
def guardar_noche(datos: dict):
    horas = datos["horas"]
    puntos = calcular_puntuacion(horas)
    conn = get_conn()
    with conn.cursor() as cursor:
        cursor.execute(
            "INSERT INTO registros (fecha,horas,puntuacion) VALUES (%s,%s,%s)",
            (datos["fecha"], horas, puntos))
        conn.commit()
    conn.close()
    return {"ok": True, "puntuacion": puntos}

Problemas conocidos y soluciones
P1
Flutter no se conecta al servidor
Causa más común: falta --host 0.0.0.0 al arrancar uvicorn
✓ Solución: siempre arrancar con uvicorn main:app --host 0.0.0.0 --port 8000. Sin ese flag solo acepta conexiones de la Pi misma.
P1
El servicio systemd no arranca
Causa: ruta del venv incorrecta en el .service, o main.py tiene errores de sintaxis
✓ Solución: sudo journalctl -u sleep-api -n 50 --no-pager para ver el error exacto. Verificar que la ruta del ExecStart es la del venv.
P2
mariadb-secure-installation da error
Causa: el comando mysql_secure_installation ya no funciona en Pi OS 64-bit moderno
✓ Solución: usar sudo mariadb-secure-installation. En pregunta 2 responder n (no unix_socket) para que Python pueda entrar con contraseña.
P2
Python no puede conectarse a MySQL
Causa: bind-address incorrecto o usuario sin permiso de conexión externa ('%')
✓ Solución: verificar /etc/mysql/mariadb.conf.d/50-server.cnf tenga bind-address=0.0.0.0 y que el usuario fue creado con 'user'@'%'.
P3
Flutter no encuentra el servidor en el teléfono
Causa: la URL usa localhost o una IP local que el teléfono no alcanza
✓ Solución: usar la IP de Tailscale (100.75.19.64:8000) o la IP local del WiFi. Nunca usar localhost desde el teléfono — ese es el teléfono mismo.
P3
App cae cuando la Pi está apagada
Causa: no hay manejo de errores de red en el FutureBuilder
✓ Solución: en el FutureBuilder agregar bloque hasError que muestre un mensaje amigable en vez de romper la app.
P4
PyMySQL error de autenticación
Causa: contraseña incorrecta o usuario sin acceso desde localhost
✓ Solución: verificar que el usuario fue creado con 'sleep_user'@'%'. El host en PyMySQL debe ser localhost porque Python corre en la misma Pi.
P4
El JSON que devuelve no coincide con lo que espera Flutter
Causa: P4 cambió un campo sin avisar a P3
✓ Solución: siempre respetar el contrato JSON acordado. Si hay que cambiar un campo, avisar al equipo y actualizar el contrato en este roadmap.
